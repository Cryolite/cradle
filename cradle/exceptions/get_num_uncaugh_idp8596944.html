Function get_num_uncaught_exceptions

cradle::exceptions::get_num_uncaught_exceptions — Returns the number of uncaught exceptions.

Synopsis

// In header: <cradle/exception/uncaught_exception.hpp>


std::size_t get_num_uncaught_exceptions();

Description

[Definition: In this description, X and Y denote evaluations[1]. Y is said to be a direct predecessor of X if Y is sequenced before[2] X and there is no other evaluation Z such that Z is sequenced before X and Y is sequenced before Z. ]

[Definition: The number of uncaught exceptions is a natural number defined for every evaluation in a well-formed program[3]. The number of uncaught exceptions of an evaluation is defined as follows;

  • 0 if the evaluation does not have any direct predecessor, otherwise
  • n + 1 if one of the direct predecessors of the evaluation is an evaluation of a throw-expression[4] and the number of uncaught exceptions for the direct predecessor is n, otherwise
  • n - 1 if the evaluation is of an expression in the compound-statement of a handler[5] and one of the direct predecessors of the evauation is not an evaluation of an expression in the same compound-statement and the number of uncaught exceptions for the direct predecessor is n.
] [6]

Returns:

 The number of uncaught exceptions.

Throws:

 Nothing.

Complexity:

 Constant.

Notes:

 Guarantees strong thread-safety.

[1] Defined in [intro.execution]/12, ISO/IEC 14882:2011.

[2] Defined in [intro.execution]/13, ISO/IEC 14882:2011.

[3] Defined in [defns.well.formed], ISO/IEC 14882:2011.

[4] Defined in [except.throw], ISO/IEC 14882:2011.

[5] Defined in [except]/1 and [except.handle]/1, ISO/IEC 14882:2011.

[6] It cannot happen that the numbers of uncaught exceptions are different among the direct predecessors of an evaluation in any well-defined program.
