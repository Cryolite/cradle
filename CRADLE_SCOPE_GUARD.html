Macro CRADLE_SCOPE_GUARD

CRADLE_SCOPE_GUARD — Provides a concise way to guarantee some statements are certainly executed upon exit from a scope.

Synopsis

// In header: <cradle/scope_guard/scope_guard.hpp>

CRADLE_SCOPE_GUARD

Description

Requires

Should be used only in the following form; 

CRADLE_SCOPE_GUARD compound-statement ;

Any other forms of use are ill-formed; no diagnostic required.

Emulated grammatical extensions to ISO/IEC 14882:2011

  • [dcl.dcl].  Insert a new syntactic category scope_guard-declaration as an alternative to the right-hand side of the production rule for block-declaration;

    block-declaration:

    • simple-declaration
    • asm-definition
    • namespace-alias-definition
    • using-declaration
    • static_assert-declaration
    • alias-declaration
    • opaque-enum-declaration
    • scope_guard-declaration

  • Add a new production rule for scope_guard-declaration;

    scope_guard-declaration:

    • CRADLE_SCOPE_GUARD compound-statement ;

Identifiers introduced into enclosing scope
  • cradle_scope_guard_ ## __LINE__
Semantics

CRADLE_SCOPE_GUARD compound-statement ;

is equivalent to 

unspecified-scope-guard-type<unspecified> cradle_scope_guard_ ## __LINE__ ([&]() noexcept compound-statement );

where 

template<typename F>
class unspecified-scope-guard-type
{
public:
  explicit unspecified-scope-guard-type(F &&f) noexcept
    : f_(std::move(f))
  {}

  ~unspecified-scope-guard-type() noexcept
  {
    f_();
  }

private:
  F f_;
};
